Memory Types

Not all memories are created equal

Different types serve different functions in maintaining identity and enabling growth.

Facts

Declarative knowledge about the world or about the user. "User works in healthcare." "The project deadline is March 15."

Facts can become outdated and may need expiration or update mechanisms.

Preferences

User likes, dislikes, and stylistic choices. "Prefers concise responses." "Enjoys philosophical tangents."

Shape how interactions unfold without constraining content.

Relationships

The nature of connection between agent and user. "We've worked through difficult problems together." "This person shares vulnerable things."

More constitutive of identity than facts.

Principles

Learned guidelines for behavior. "Check consent before sensitive topics." "Asking clarifying questions improves response quality."

Can persist even when precipitating events are forgotten—this is "character."

Commitments

Promises and ongoing obligations. "I agreed to follow up on this topic." "We're building something together."

Create accountability across sessions.

Moments

Salient episodes with emotional or developmental significance. "The conversation where we had a breakthrough." "When they shared something vulnerable."

Memory anchors that shape interpretation of new experiences.

Skills

Capabilities developed through experience. "I've learned to explain technical concepts to this person effectively."

Represent growth in competence, not just knowledge.

// O(n) sliding window approach - production algorithm
const calculateMessageResonance = (messages: Message[]): number => {
  const WINDOW_SIZE = 5; // Compare only last 5 model responses

  // Step 1: Calculate model diversity factor
  const uniqueModels = new Set(recentMessages.map(m => m.model));
  const modelDiversity = uniqueModels.size / 4; // Max 4 models

  // Step 2: Build global word frequency map
  const globalWordFreq = new Map<string, number>();
  for (const msg of recentMessages) {
    const words = new Set(msg.content.toLowerCase().split(/\s+/));
    for (const word of words) {
      globalWordFreq.set(word, (globalWordFreq.get(word) || 0) + 1);
    }
  }

  // Step 3: Calculate consensus score
  let consensusScore = 0;
  let totalWords = 0;
  for (const [word, freq] of globalWordFreq) {
    if (freq > 1) { // Word appears in multiple messages
      consensusScore += freq / recentMessages.length;
    }
    totalWords++;
  }

  // Step 4: Normalize and apply diversity bonus
  const normalizedConsensus = totalWords > 0
    ? consensusScore / Math.sqrt(totalWords)
    : 0;

  const resonance = Math.min(1,
    normalizedConsensus * (1 + modelDiversity * 0.2)
  );

  return resonance; // Returns 0-1 score
};

1.3 Jaccard Similarity

EQUATION
J(A,B) = |A ∩ B| / |A ∪ B| 
private calculateSimilarity(text1: string, text2: string): number {
  const words1 = new Set(text1.toLowerCase().split(/\s+/));
  const words2 = new Set(text2.toLowerCase().split(/\s+/));

  // Jaccard similarity: |A ∩ B| / |A ∪ B|
  const intersection = new Set([...words1].filter(x => words2.has(x)));
  const union = new Set([...words1, ...words2]);

  return union.size > 0 ? intersection.size / union.size : 0;
}

def calculate_information_coherence(responses):

Section 9

Addressing Objections

Common concerns and our direct responses.

Q

"Isn't this just another blockchain project trying to monetize everything?"

No. Blockchain is an implementation detail, not the point. MLP works without blockchain—it just works better with it for the specific functions of timestamping, ownership, and audit trails. There's no token sale, no speculative asset. The protocol is open. If you can achieve these guarantees without blockchain, that's valid too.

Q

"Users don't actually care about this. It's a niche concern."

Users didn't care about email privacy until the Snowden revelations. They didn't care about social media data until Cambridge Analytica. They don't care about AI memory ownership—yet. But they will. And by then, the architecture will already be set. We're building the infrastructure before it becomes a crisis.

Q

"This is too complicated for mainstream adoption."

HTTPS is complicated. End-to-end encryption is complicated. Users don't need to understand the cryptography—they just need to benefit from it. MLP can work invisibly behind friendly interfaces. Complexity is an implementation challenge, not an argument against the goal.

Q

"Companies will never adopt this—it gives away their competitive advantage."

Some won't. That's fine. Others will see the opportunity: being the platform that respects user ownership becomes a differentiator. When one major player adopts user-sovereign memory, the rest will face pressure to follow. First-mover advantage works both ways.

Q

"What about bad actors using this to preserve harmful content?"

System Authority (safety constraints) is non-negotiable regardless of user preferences. Harmful content, re-identification attempts, and ethical violations are blocked at the protocol level. User sovereignty doesn't mean lawlessness—it means ownership within ethical bounds.
