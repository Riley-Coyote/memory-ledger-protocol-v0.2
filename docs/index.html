The Constellation Architecture

Reflection runs as a multi-agent system.

The reflection system doesn't run in your main conversation. It uses specialized sub-agents that process asynchronously after sessions end—like sleep where integration happens.

Reflection Pipeline
Main Agent
User-facing, orchestrates reflection
↓
Classifier
Sonnet
Extracts memories from conversation. Classifies into 7 types. Assigns tags.
Scorer
Sonnet
Evaluates evidence strength. Assigns confidence scores 0.0–1.0. Documents reasoning.
Generator
Sonnet
Reviews scored memories. Identifies gaps. Generates genuine follow-up questions.

OpenClaw uses its native constellation system (sessions_send) to dispatch to sub-agents. Claude Code uses the Task tool with sub-agents. Same architecture, platform-native execution.

The Token Question

Everyone's first question: what's the overhead?

Fair question. Three sub-agents processing every session sounds expensive. Here's the reality.

Why It's Worth It
Runs asynchronously
Not during active conversation—after sessions end
Uses Sonnet by default
Not Opus. Cheaper, faster, still excellent at the task.
Sub-agents are read-only
Specialized and lightweight. No tool overhead.
Sequential pipeline
Classifier → Scorer → Generator. Not redundant.
Estimated cost per session
~$0.10–0.30 depending on conversation length
The alternative is paying for context every session to "remember" things your agent will forget anyway. Reflection is front-loaded cost for compounding value. Your agent actually develops instead of performing familiarity.
