Architecture

What lives where.

The ledger is public but reveals nothing. Storage holds content but can't read it. You hold the keys that make any of it meaningful.

Public
On the Ledger
  • Memory envelope IDs
  • Content-addressed pointers
  • Integrity hashes
  • Timestamps
  • Attestation signatures
  • Access policy references
  • Lineage records
Encrypted
In Storage
  • Actual memory content
  • Structured metadata
  • Embeddings & features
  • Symbolic compressions
  • Derived artifacts
Yours
Your Keys
  • Decryption keys
  • Signing keys
  • Access grants
  • Revocation power
  • Identity kernel

Coordination

Why a token exists.

Not ideology. Economics. Someone has to run the infrastructure without recreating centralized control.

01
Payment
Storage nodes, verification nodes, and protocol development get paid for work done.
02
Governance
Token holders vote on protocol upgrades and standards. No single company controls direction.
03
Alignment
If you hold the token, you want the network to work. Everyone has skin in the game.
Consequences

What this enables.

Platform Migration
Switch AI providers without losing context. Export your memory, import it elsewhere, continue where you left off.
Multi-Platform
Use different AI systems for different tasks. They all draw from your shared memory pool. Context follows you.
True Deletion
Crypto-shred: destroy the keys and the ciphertext becomes permanently unreadable. Stronger than "please delete my data."
Inheritance
Specify what happens when you die: transfer to heirs, donate to researchers, or shred everything. Your choice, enforced by math.
The Guardian
An AI that knows your cognitive rhythms and works for you — not engagement metrics. Requires sovereign memory to exist.
